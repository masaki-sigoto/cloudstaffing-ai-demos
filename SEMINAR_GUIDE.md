# セミナー実演ガイド

4本のデモをセミナーで実演するためのチェックリストと進行手順。

## 事前準備チェックリスト

### 環境確認
- [ ] Python 3.10 以上が入っている（`python3 --version`）
- [ ] `ai-demos/` ディレクトリ直下で Claude Code を起動できる
- [ ] Chromeなど最新ブラウザが起動できる
- [ ] プロジェクタ/画面共有設定済み（ターミナルとブラウザを切替）

### プロジェクト別の事前実行
各プロジェクト初回のみ事前に1回実行しておくと、実演時の待ち時間を減らせる。

```bash
# 01: out/ に生成物がすでにある状態で実演開始
cd ai-demos/01_csv-automation && python3 src/main.py convert --input samples/timesheet_202604_haken_a.csv

# 02: PyYAML の有無を事前に確認（なければ内蔵パーサで動く）
cd ../02_rounding-checker && python3 src/main.py validate --config samples/rules/employee_friendly.yml

# 03: generate-samples を一度走らせておく
cd ../03_attendance-check && python3 src/main.py generate-samples --count 12

# 04: ブラウザで一度開いて動作確認
open ../04_yesno-diagnosis/src/index.html
```

## 実演フロー

### 冒頭（3分） — 課題提起
- 「月初月末の派遣管理業務、どれだけ手作業でしょうか？」
- Excel・紙・確認地獄・差戻しの往復の現状共有
- 「今日は4つのAI活用アイデアを実際にターミナルとブラウザで見せます」

### 01 CSV加工（5分） — 全企業に効く汎用性の提示
1. **Before**（1分）: 派遣元から届くCSVを画面表示 `cat samples/timesheet_202604_haken_c.csv`
   - 列順バラバラ、全角半角混在、令和表記、エラー行混入
   - 「これを毎月毎社から受け取り、1件ずつ手直し…」
2. **After**（1.5分）: `/csv-convert samples/timesheet_202604_haken_c.csv`
   - 一発で標準形式に整形、エラー行はハイライト
   - 入出力件数・要確認件数が可視化
3. **テンプレート再利用**（1分）: `/csv-save-template` → 翌月CSVに適用
   - 「同じ派遣元からの来月のCSVは、設定ゼロで自動整形」
4. **締め**（1分）: 派遣元メリット（加工工数減）/派遣先メリット（差戻し減）の再強調

### 02 端数処理（7分） — 説明責任の可視化
1. **導入**（1分）: 「端数処理は派遣元/派遣先で揃わないまま、請求ズレの温床」
2. **基準計算**（1分）: `python3 src/main.py simulate --config samples/rules/strict_1min.yml --punch 9:03,18:07 --hourly 1800`
   - 1分単位での素の計算結果
3. **スタッフ有利ルール**（1.5分）: `--config samples/rules/employee_friendly.yml` で実行
   - 金額増、増加方向の警告も表示
4. **衝撃比較**（2分）: `/rounding-compare` 3ルールを並べる
   - 「同じ打刻で **1日900円・月18,000円** 違う」
5. **explain で説明責任**（1.5分）: `/rounding-explain --demo` で「なぜこの結果か」を3行で
   - 「派遣先への説明、派遣スタッフへの給与説明、両方ラクに」

### 03 勤怠チェック（9分） — CSワークフローへの深い結合
1. **3者ワークフロー共有**（1.5分）: スタッフ打刻→派遣先承認→派遣元集計→月次締め、の流れを図示
2. **従来の確認地獄**（1分）: `wc -l samples/202604/timesheet.csv` で「毎月266件を人の目で…」
3. **AI絞り込み**（2分）: `/attendance-check`（または `/attendance-demo`）
   - 「266件 → 要確認20件（高:11 中:8 低:1）」
4. **担当者別通知**（1.5分）: `cat out/notifications/U-001_sato.txt`
   - 「派遣元コーディネーター佐藤さんが見るべき、自分の担当スタッフ6件」
5. **派遣先事業所別リスト**（1.5分）: `cat out/checklist/by_client_site_202604.txt`
   - 「派遣先承認者への再承認依頼資料にそのまま使える」
6. **締め**（1.5分）: `cat out/run_summary_202604.json | python3 -m json.tool`
   - 3軸スコアリング（給与・請求・法令）、再現性（--as-of-date）、データガード機構の説明

### 04 YES/NO診断（8分） — 営業・導入支援で即戦力
1. **使いどころ共有**（1分）: 「営業の初回商談、導入支援の初回ヒアリング」「属人化解消」
2. **ショート版 5分体験**（3分）: ブラウザで7問に回答
   - 想定動線: Q2=Y→Q3=Y→Q5=Y→Q12=Y→Q13=労使協定→Q6=N→Q15=Y
   - → **パターンC（将来拡張対応型）** に着地
3. **結果画面の解説**（1.5分）: 推奨事項、要ヒアリング項目、営業メモ欄
4. **印刷（匿名化）**（1.5分）: Ctrl+P → 要約版/詳細版
   - 「派遣元担当者に渡せる・社内共有にも使える」
5. **締め**（1分）: パターンA〜E の意味、導入前フェーズでの活用先

### まとめ・質疑（3分 + 質疑）
- 4本の共通テーマ: 「派遣管理業務の AI 化は、一気通貫より **点で効く** 4本のような小さい AI を積む方が現実的」
- 社内での内製検討・運用提案へのつなぎ
- 質疑応答

## 質疑で想定される質問と回答の糸口

- **「学習データはどこから？」** → 今回は AI API 不使用のルールベース実装。同義語辞書・スコアリング・決定表で「AI らしさ」を演出。実運用では LLM 統合も可能だが、まず現状の運用課題に効く粒度から。
- **「セキュリティは？」** → データはローカル完結、外部通信ゼロ、PII マスキング実装済。04 はブラウザ完結（localStorage もデフォルトOFF）。
- **「導入コストは？」** → 本日見せたのはモック（数日レベル）。本番化は業務の擦り合わせが本体で、技術コストは軽め。
- **「派遣法の解釈は？」** → 本デモの診断は整備状況把握の補助ツール。法的判断は社労士等の専門家へ。

## トラブルシュート

### Python が `python3: command not found`
- macOS 標準は `python3`。Homebrew: `brew install python@3.11`

### PyYAML が無い警告（02）
- 内蔵 mini-parser で動くので続行可。気になれば `pip install --user pyyaml`

### 04 の HTML が開かない
- `open 04_yesno-diagnosis/src/index.html` または Finder でダブルクリック
- ブラウザが警告を出した場合は「開く」を選択

### 実行が遅い・時間がかかる
- 01 は 10,000 行で 5 秒以内を目標。サンプルは 5〜10 行なので即時
- 03 は `generate-samples --count 12` で 266 行、処理時間 0.02〜0.05 秒
