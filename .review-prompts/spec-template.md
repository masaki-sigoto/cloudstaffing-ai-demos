# 技術仕様書テンプレート

※ 設計書の「どう作るか」に対し、技術仕様書は「コードに落とす直前の具体形」を定義する。関数シグネチャ、型、データスキーマ、アルゴリズム近似コード、依存バージョン、テスト設計までを明記し、実装者が迷わず着手できる粒度に仕上げる。

---

## 1. 仕様書の位置付け
- 要件定義書・設計書との対応
- この仕様書がカバーする範囲とカバーしない範囲
- 実装着手の前提条件

## 2. 動作環境と依存
- Python バージョン（最低要件）
- 依存ライブラリ（バージョン固定、標準ライブラリ優先方針）
- OS / ロケール / 文字コードの前提
- （HTMLプロジェクトの場合）対応ブラウザ、file:// 動作、外部CDN禁止

## 3. ディレクトリ構成（実装版）
```
project-root/
├── src/
│   ├── __init__.py
│   ├── main.py
│   └── ...（具体的なモジュール配置）
├── samples/
├── out/
├── tests/
└── README.md
```
各ファイル/ディレクトリの役割を1行で。

## 4. データスキーマ（実装レベル）
- 入力ファイル（CSV/JSON/YAML）の列定義、型、制約
- 出力ファイルの列定義、型、制約
- 内部データモデル（クラス/dict 構造）
- スキーマバージョン管理（必要なら）

## 5. 関数/クラス仕様
### 5.x ModuleName
- **責務**: （1行）
- **公開API**:
  ```python
  def function_name(arg1: Type, arg2: Type = default) -> ReturnType:
      """1行説明"""
  ```
- **例外**: 投げる可能性のある例外クラス
- **副作用**: ファイルI/O、環境変数、stdout/stderr 等

## 6. アルゴリズム詳細
主要ロジックを疑似コード or near-Python で。設計書より具体。
- 分岐、ループ、例外ハンドリング
- 計算式の実装形
- データ変換の具体ステップ

## 7. CLI/UI 仕様（実装版）
### CLI の場合
- サブコマンド体系（argparse 構造）
- 各オプションの型、デフォルト、バリデーション
- 終了コード（0/1/2/...）の意味
- 標準出力/標準エラー出力の出し分け

### UI の場合
- DOM 構造の骨格
- イベントハンドラ
- ARIA 属性方針
- CSS クラス設計（主要のみ）

## 8. エラー処理と例外階層
```python
class DemoError(Exception): pass
class InputValidationError(DemoError): pass
class RowCountMismatchError(DemoError): pass
...
```
- 各例外の発生条件
- 終了コードマッピング
- エラーメッセージ規約

## 9. ログ仕様
- ログレベル定義（DEBUG/INFO/WARN/ERROR）
- ログフォーマット（テキスト例）
- 出力先（stdout/stderr/file）
- PII マスキング適用箇所

## 10. テスト設計
- 単体テスト対象モジュール一覧
- 重要テストケース（入力 → 期待出力）の代表例
- ハッピーパス / エッジケース / 異常系
- サンプルデータとの対応
- （最小実装前提で、テスト自動化はMAY）
