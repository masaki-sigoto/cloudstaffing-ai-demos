# /attendance-demo

セミナー実演用。派遣元担当者・派遣先承認者・スタッフの3者ワークフローで発生する月次締め前の確認工数を「約10倍短縮」する演出を行う。

## 演出シナリオ（7〜10分）

### Step 1: 状況説明（1分）
派遣元「クロスリンクスタッフ」の勤怠チェック担当者 田中さんは、4月度の月次締め日（2026-04-30）の2営業日前に、担当スタッフ全件の勤怠を確認する必要がある。派遣先3社（ABC商事・XYZ物流・DEF製造）の承認状態と、スタッフからの休暇・残業申請の整合性を、248件全件目視で確認している。

### Step 2: 実行（30秒）
```bash
cd "/Users/apple/Library/Mobile Documents/com~apple~CloudDocs/管理フォルダ/01_会社別/クロスリンク/06_クラウドスタッフィング/ai-demos/03_attendance-check"
python3 src/main.py check --month 2026-04 --as-of-date 2026-04-28 --data-class dummy
```

### Step 3: サマリ強調（30秒）
画面で強調されるサマリ：
```
全 NNN 件 → 要確認 M 件
    [高] X 件 / [中] Y 件 / [低] Z 件
```
演者コメント：「248件を目視で全件見ていたのが、AIが拾ったこの M 件だけでいい。しかも高優先度の4件だけ見れば月次締めに進めます」

### Step 4: 3者ワークフローの具体性を解説（2分）
- 派遣元コーディネーター別通知（`out/notifications/U-001_sato.txt`）：担当スタッフの対応必要件数
- 派遣先事業所別チェックリスト（`out/checklist/by_client_site_*.txt`）：派遣先承認者への差戻し資料
- 各Findingに「推奨アクション」が付いており、どの異常を誰に照会すべきかが明確

### Step 5: 10異常パターンの網羅性を見せる（2分）
- A-01 退勤打刻漏れ、A-02 出勤打刻漏れ
- A-03 休憩未入力、A-04 連続24時間超勤務
- A-05 1日複数回の出退勤
- A-06 申請×実績不整合（休暇×打刻矛盾・残業申請漏れ）
- A-07 派遣先承認待ち滞留、A-08 深夜打刻（シフト外）
- A-09 シフト大幅乖離、A-10 重複打刻

### Step 6: 締め（1分）
「派遣元→派遣先→スタッフの3者間の差戻しで最も負荷が高いのが月末。締め日の作業がこれだけ楽になる、というのが今日一番お伝えしたいことです」
