## 🧭 派遣業界論点 1 ページ要約（このデモが効く位置づけ）

派遣管理業務は **3 者構造**（派遣スタッフ／派遣元／派遣先）で動き、特に月次締め前後に確認作業が集中します。本デモは以下の論点に対して「点で効く」 4 本を提示します。

- **3 者構造**: スタッフ打刻 → 派遣先承認者の承認 → 派遣元集計 → 月次締め。情報の往復が多く、不一致が起きやすい（→ 03 勤怠チェックが効く）。
- **月次締め前のピーク**: 締め日直前に勤怠・請求・契約の確認が集中。整形・抽出・並び替えの定型作業を AI が支援する余地が大きい（→ 01 CSV加工 / 02 端数処理）。
- **法令観点**: 派遣業務には複数の法令論点（同一労働同一賃金、抵触日、36 協定など）が前提として存在する。抵触日は **原則** 事業所単位 3 年・個人単位 3 年だが、**例外** として労使協定や対象業務など条件によって取扱いが異なる場合がある。最終的な法的判断は社労士等の専門家への確認が前提で、整備状況の把握自体は内製で可能（→ 04 運用整備診断）。
- **正確性の担保**: 端数処理など正確性が必要な計算はロジック処理（真理値表）で実装し、AI の自由判断を入れない設計（→ 02 端数処理）。

各デモの詳細は [SEMINAR_GUIDE.md](./SEMINAR_GUIDE.md) と各プロジェクトの `docs/01_requirements.md` を参照してください。
