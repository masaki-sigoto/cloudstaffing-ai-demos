# yesno-demo

セミナーでの実演シナリオをガイドします。**営業シナリオA（5分）** と **導入支援シナリオB（10分）** の2パターンです。

## シナリオA: 営業商談の初回（5分・ショート版）

### 設定
セミナー登壇者が営業役となり、架空の派遣会社「カゴシマ派遣株式会社」の社長役と商談している体。

### 実演フロー

1. **[0:00]** アプリを開く → スプラッシュ画面
   - 「保存しない（一時利用）」が初期選択されていることを確認
   - **「ショート版で開始（7問／約5分）」** をクリック

2. **[0:30 - 3:30]** 7問に回答
   | 質問 | 回答（カゴシマ派遣） |
   |------|---------------------|
   | Q2 拠点3箇所以上？ | **はい** |
   | Q3 拠点ごとに承認者？ | **はい** |
   | Q5 本社で一元管理したい？ | **はい** |
   | Q12 3年超の派遣契約扱う？ | **はい** |
   | Q13 同一労働同一賃金の方式？ | **労使協定方式** |
   | Q6 代理承認ルート決めている？ | **いいえ** |
   | Q15 今後1年で拠点追加計画？ | **はい** |

3. **[3:30]** 診断結果カード表示
   - **パターンC: 将来拡張対応型** が表示される
   - 要ヒアリングバッジ: 「代理承認ルート未整備(Q6)」「省略項目あり: Q1/Q4/Q7/Q8/Q9/Q10/Q14/Q16/Q17」
   - 推奨設定サマリ5項目、設定工数目安（10〜15営業日）、注意点

4. **[4:30]** 「要約版を印刷」ボタンクリック → A4 1枚の印刷プレビュー
   - ブラウザの印刷で「PDFとして保存」→ 顧客にメール送付の流れ

### キートーク
- 冒頭: 「派遣管理の運用って、契約・勤怠・請求・法対応と論点が多くて、どこまで整っているかの棚卸しが一番ややこしいところなんですよ。」
- 結果提示: 「御社は **将来拡張対応型** ですね。拡張計画があるので、最初から3層構造で設計しておくと後々の手戻りが減ります。」
- 印刷: 「このシート、印刷してお渡ししますね。」

## シナリオB: 導入支援のキックオフ（10分・フル版）

### 設定
契約後、CS導入支援担当がオンラインMTGで顧客の情シス担当と画面共有。

### 実演フロー

1. スプラッシュ → **「フル版で開始（最大15問／約10分）」**
2. Q1 〜 Q17 まで順に回答（分岐により最大15問）
3. 診断結果 → **「詳細版を印刷」** ボタン
   - 「設定項目チェックリスト」（カテゴリ／項目／状態）が出る
   - 回答一覧が全質問分出る
4. 営業メモ欄に「次回までに準備いただく項目」をメモ
5. 匿名化トグル OFF + 営業メモ印字 ON にして詳細版を印刷

### クロージング
- 「次のMTGまでに、この『要ヒアリング』の赤字項目の情報だけご準備いただけますか？」で締める
- 詳細版PDFを共有フォルダに保存する流れをデモ
