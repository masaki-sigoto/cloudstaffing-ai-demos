# 運用整備ナビゲーター（04_yesno-diagnosis）

> クラウドスタッフィング（派遣管理SaaS）の **導入前ヒアリング補助／派遣管理運用の整備状況簡易診断** アプリ。
> 単一HTMLファイル完結、ダブルクリックで起動、オフライン動作。

---

## これは何？

派遣元企業の派遣管理運用（**契約／案件／勤怠／請求／承認フロー／法対応／データ運用**）の整備状況を、YES/NO の質問に答えるだけで診断し、**5つの運用整備パターン** のうち1つに分類するツールです。

- **営業フェーズ**: 商談の初回ヒアリング補助として5分で診断結果を提示
- **導入支援フェーズ**: キックオフMTGの標準ヒアリングシートとして10分で全項目ヒアリング
- **セミナー枠**: AI活用デモの特別枠として、ビジュアル映えする診断結果カード+印刷ヒアリングシート

## 起動方法

**`src/index.html` をダブルクリックするだけ。**

- ブラウザでURLを開く必要なし
- サーバー不要、ビルド不要、ネット接続不要
- Chrome / Edge / Safari / Firefox の最新版で動作（IE非対応）
- ファイルサイズ 約80KB（目標300KB／上限1MB）

```
/Users/.../04_yesno-diagnosis/src/index.html  ← ダブルクリック
```

## 5分ショート版 vs 10分フル版の使い分け

| 版 | 質問数 | 所要時間 | 使うシーン |
|----|--------|---------|-----------|
| **ショート版** | 7問（固定順）| 約5分 | 営業の初回商談、セミナーデモ、「とりあえず御社はどのタイプか」を素早く出したいとき |
| **フル版** | 最大15問（分岐あり）| 約10分 | 導入支援のキックオフMTG、契約後の本格ヒアリング、詳細チェックリストが必要なとき |

ショート版は `Q2 → Q3 → Q5 → Q12 → Q13 → Q6 → Q15` の固定ルート。フル版は `Q1` から分岐ツリーに従い Q17 まで進みます。

## パターンA〜E の意味

本アプリは診断結果を以下5種の **運用整備パターン** に分類します。
※「整備段階」を表すもので、優劣ではありません。

| パターン | 名称 | 該当条件 | 想定企業 |
|---------|------|---------|----------|
| **A** | 基本整備型 | 単一拠点・承認1段階・法対応基本 | 従業員30名未満、単一オフィスの派遣会社 |
| **B** | 複数拠点対応型 | 少拠点(1〜2)・承認2段階 | 本社＋営業所1〜2、従業員30〜100名の中堅 |
| **C** | 将来拡張対応型 ★デモの主役 | 多拠点(3+)・承認2〜3段階・拡張計画あり | 全国展開の中堅〜大手、M&A志向 |
| **D** | 多事業所分散型 | 多拠点・本社集約しない・拠点別ルール | 地域色の強い派遣会社、フランチャイズ型 |
| **E** | 法対応要整備型 | 36協定未整備 OR 抵触日運用未整備 OR 代理承認未整備 | 派遣法関連の運用体制に未整備要素を含む企業 |

### 判定の優先順位（§10.3.2 決定表）

1. **E**（法対応要整備：C=3）
2. **C**（L=3 かつ A>=2 かつ 拡張計画あり）
3. **D**（L=3 かつ 本社集約せず／拠点別ルール）
4. **B**（L=2 かつ A=2）
5. **A**（L=1 かつ A=1 かつ C=1）
6. フォールバック（最近傍パターン・確定度：中）

## セミナー実演シナリオ

### シナリオA: 営業商談の初回（5分デモ）

1. 登壇者が営業役で、架空の派遣会社「カゴシマ派遣株式会社」の社長役と商談を始める
2. アプリを開き、**「ショート版で開始」** をクリック
3. 7問に1問30秒ペースで回答（想定回答はパターンC へ着地する設計）
   - Q2=YES, Q3=YES, Q5=YES, Q12=YES, Q13=労使協定方式, Q6=NO, Q15=YES
4. **診断結果カード: パターンC 将来拡張対応型** が表示される
5. 要ヒアリングバッジ: 「代理承認未整備」「省略項目あり」が黄色で出る
6. **「要約版を印刷」** ボタン → A4 1枚のヒアリングシートを「手土産」として提示

キートーク例:
- 「派遣管理の運用って論点が多くて、棚卸しが一番ややこしいところなんですよ。5分だけください」
- 「御社は **将来拡張対応型** ですね。拡張計画があるので3層構造で最初から設計すると後の手戻りが減ります」
- 「このシート、印刷してお渡ししますね」

### シナリオB: 導入支援のキックオフ（10分デモ）

1. 契約後、CS導入支援担当がオンラインMTGで顧客の情シス担当と画面共有
2. **「フル版で開始」** をクリック
3. Q1〜Q17 まで順に回答（分岐により最大15問）
4. 診断結果 → **「詳細版を印刷」**
   - 設定項目チェックリスト（カテゴリ／項目／状態）が表形式で出る
   - 回答一覧（質問ID・質問文・回答・内部コード・業務領域）が全問分出る
   - マッチしたルール（優先度・ルールラベル）が明示される
5. 営業担当者向けメモ欄に「次回までの宿題」を追記
6. 「匿名化印刷 OFF」+「営業メモを印字する ON」にして詳細版を印刷

クロージング例:
- 「次のMTGまでに、この『要ヒアリング』項目の情報だけご準備いただけますか？」
- 詳細版PDFを共有フォルダに保存する流れをデモ

## 主な機能

- ✅ YES/NO の動的分岐（フル版15問、ショート版固定7問）
- ✅ 進捗バー（フル版は回答により到達予定が変動）
- ✅ 1問前に戻る（Y/N キー、← キー対応）
- ✅ 5パターン判定（L/A/C/P 軸スコア＋決定表）
- ✅ 要ヒアリングタグ自動付与（黄色バッジで視覚化）
- ✅ 診断結果カード（パターン名／色分け／推奨設定5項目／工数／注意点）
- ✅ 印刷2種（要約版A4 1枚／詳細版A4 2〜3枚、全ページに免責反復表示）
- ✅ 匿名化印刷トグル（企業名・担当者名を伏せ字化、デフォルトON）
- ✅ 営業メモ印字サブトグル（詳細版のみ、デフォルトOFF）
- ✅ 結果サマリをクリップボードへコピー（メール貼付用）
- ✅ 保存しないモード（デフォルト）／保存するモード（明示オプトイン）
- ✅ `localStorage` 不可環境でのフォールバック（黄バナー＋セッションメモリ動作）

## プライバシー

- **回答データは一切サーバーに送信しません**（外部通信ゼロ）
- 保存モードは **「保存しない」が初期デフォルト**（安全側）
- 企業名・担当者名・営業メモは **localStorage に保存しません**（セッションメモリのみ）
- 印刷プレビューは **匿名化印刷ONが初期デフォルト**（企業名・担当者名を伏せ字化）
- 「顧客情報漏洩リスクなし」のような断定表現は使用しません

## 免責

本ツールは **診断補助** であり、**法令判定ではありません**。派遣法・労基法等の法令適合性判定は行いません。画面上に表示される推奨設定・法対応フラグは「要確認項目の洗い出し」であり、最終判断は顧客側の法務／労務専門家によるレビューを前提とします。

## 開発メモ

- Vanilla JS のみ（React/Vue/jQuery 不使用）
- IIFE + 名前空間でモジュール分割（State/Flow/Judge/Render/Persist/AppCore）
- PATTERN_RULES を判定の単一ソース（predicate 関数で記述）
- 起動時に `validateConfig()` で設定整合をチェック
- 印刷は `position: fixed` + `body margin-top 14mm` で全ページ反復

詳しくは `docs/03_spec.md` を参照。
